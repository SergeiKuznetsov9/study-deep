Теперь переведем конфигурацию webpack на TypeScript. Для этого установим пакеты типов
npm i -D ts-node@10.5.0 @types/node@17.0.21 @types/webpack@5.28.0

а после поменяем расширение конфига вебпака на ts

Когда мы начнем менять импорты с коммонJS на екмуСкрпит, у нас будут ошибки.
Мы можем импортировать все при помощи * (import * as webpack from "webpack";)

Чтобы импорты были более привычными, нам необходимо добавить пару настроек 
в конфиг тайпСкрипта:
  "compilerOptions": {
    "allowSyntheticDefaultImports": true,
    "esModuleInterop": true
  }

Далее, в конфиге вебпака обозначим переменную и присвоим ей весь конфиг. Переменной
укажем тип, чтобы можно было использовать автокомплит

После всех манипуляций работать ничего не будет. Это происходит потому, что ts-node
не поддерживается никакой синтаксис модуля, кроме commonjs (это взято из доки webpack).

Для решения в ТСКонфиг добавим следующую настройку:
"ts-node": {
    "compilerOptions": {
      "module": "CommonJS"
    }
  }
