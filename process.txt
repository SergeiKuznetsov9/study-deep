Для анализа сборки воспользуемся плагином webpack-bundle-analyzer. Установим его:
npm install --save-dev webpack-bundle-analyzer

Импортируем его и вкинем в массив конструктор. Также установим типы, чтобы TS не ругался
на импорт:
npm i --save-dev @types/webpack-bundle-analyzer

После добавления этого плагина при старте приложения также будет открываться отдельная вкладка с
интерфейсом для анализа бандла

Однако для получения реальных данных нужно сбилдить проект. Размеры банлов будут отличаться в дев
режиме и проде.

Для того, чтобы всякий раз окошко не открывалось, добавим настройку в вебпак
  new BundleAnalyzerPlugin({
    openAnalyzer: false,
  }),
