В этом уроке добавим обертку errorBoundary. Образец возьмем в документации.

Интересный момент - то что нам в данном компоненте нужно также использовать i18n.
Но компонент явояется классовым. Чтобы i18n можно было в нем использовать, можно применить
несколько приемов.

Например мщжно применить обертку (HOC) withTranslation. Она импортируется из i18n и добавляется
при экспорте:

export default withTranslation()(ErrorBoundary)


Но мы сделаем по другому, мы будем возвращать функциональный компонент, в котором сделаем все что
захотим)

Для тестирования появления ошибки в App можно добавить useEffect:

  useEffect(() => {
    if (Math.random() > 0.5) {
      throw new Error();
    }
  }, []);
